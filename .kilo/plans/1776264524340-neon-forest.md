# 实施计划：连接 Basys3 的 UART

## 目标
让 Basys3 的 UART 串口在电脑终端中稳定显示 FPGA 输出（当前工程预期为 `123456789...`）。

## 前提与现状
- 工程约束已将 UART 管脚映射到 Basys3 板载 USB-UART：
  - `uart_tx` -> `B18`
  - `uart_rx` -> `A18`
- Basys3 板载 USB-JTAG/UART 桥接芯片会通过 micro-USB（PROG/UART 口）同时提供下载与串口功能。

## 执行步骤
1. **修改代码引入 LED 指示（可选，为了调试方便）**
   - 既然综合通过了但依然没有输出，我们可以将 `uart_tx` 绑到 LED 上直观查看。
   - `led[15]` 已被用作 `host_write_seen`（即 CPU 是否触发了 UART）。
   - 我们将 `led[14]` 映射为物理的 `uart_tx` 状态（TX 传输时拉低，即平时亮，发送时闪灭）。
   - 将 `led[13]` 映射为物理的 `uart_rx` 状态（RX 接收时拉低）。

2. **硬件连接**
   - 使用 micro-USB 数据线连接 Basys3 的 `PROG/UART` 口到电脑。
   - 给板子上电（USB 供电即可）。

2. **下载比特流**
   - 在 Vivado Hardware Manager 中连接设备并下载 `.bit`。
   - 确认下载后 FPGA 开始运行（LED/数码管有活动）。

3. **识别串口设备**
   - Windows 打开设备管理器，查看新增 `USB Serial Port (COMx)`。
   - 记下 `COMx` 端口号。

4. **终端配置**
   - 打开串口工具（PuTTY / MobaXterm / Tera Term / `pyserial`）。
   - 参数设置为：`115200`, `8 data bits`, `no parity`, `1 stop bit`, `no flow control`（即 115200 8N1）。

5. **验证输出**
   - 连接后观察终端输出，预期连续字符流：`123456789123456789...`。
   - 同时数码管应循环显示 `1..9`，两者行为一致。

## 故障排查
- **无串口设备**：更换数据线（避免仅充电线）、重插 USB、安装 Digilent/Adept 驱动。
- **端口被占用**：关闭 Vivado 串口占用窗口或其他终端程序后重连。
- **乱码**：再次确认终端是 `115200 8N1`，并确认工程 UART 分频与时钟一致（当前应按 12.5MHz 计算）。
- **完全无输出**：先看数码管是否在跑 `1..9`；若数码管正常但串口空，优先排查 COM 口号和终端连接对象是否正确。

## 交付结果
完成后可通过单根 USB 线实现：
- 下载 FPGA 程序；
- 在电脑串口终端实时接收 Basys3 的 UART 输出。
