# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Build Commands

```bash
make unit-tests              # Run all unit tests (alu, control_unit, timer, gpio, etc.)
make integration-tests        # Run rv32i integration test
make isa-regression          # Run all rv32ui ISA tests
make cocotb-tests MODULE=<name>  # Run cocotb JTAG tests (from root or tb/cocotb/)
make -C tb/uvm sim           # Run pyUVM JTAG simulation (requires Docker)
make clean                   # Clean build artifacts
```

**Single test commands:**
```bash
make _unit_alu               # Single unit test
make isa-test TEST=rv32ui-p-add  # Single ISA test
cd tb/cocotb && make MODULE=test_tap_fsm  # Single cocotb test
```

**Docker environment (recommended for pyUVM):**
```bash
docker compose build          # Build Docker image
docker compose run --rm sim   # Interactive dev shell
docker compose run --rm sim make -C tb/uvm sim  # Run pyUVM directly
```

## Architecture

### CPU Core — `rtl/riscv_core.v`
Single-cycle RV32I processor with:
- 5-stage datapath: PC → Fetch → Decode → Execute → Memory → Writeback
- Memory-mapped I/O at upper addresses:
  - `[0xFFFF0000-0xFFFF000F]` — Timer
  - `[0xFFFF0010-0xFFFF001F]` — GPIO
  - `[0x80002000]` — tohost (for riscv-tests completion signaling)
- Debug stall interface for JTAG access

### JTAG Debug Interface — `rtl/jtag_tap.v` + `rtl/riscv_core_jtag.v`
IEEE 1149.1 compatible TAP controller:
- `jtag_tap.v` — TAP state machine, IR/DR shift registers, IDCODE/BYPASS/DBGACC/DBGRST instructions
- `riscv_core_jtag.v` — Wraps `riscv_core` + decodes JTAG debug accesses:
  - `addr[4:0]` → GPR x0–x31
  - `addr[12:7]=0x20` → PC read
  - `addr[12:7]=0x21` → STATUS (halted flag)
  - `addr[12:7]=0x22` → RESUME (write=1 to resume CPU)
  - `debug_type=1` → Memory access at `addr[12:2]` (byte offset)

### RTL Module List
| Module | Purpose |
|---|---|
| `alu.v` | Arithmetic/logical operations |
| `alu_control_unit.v` | Decodes alu_op + funct3 → alu_control signal |
| `control_unit.v` | Decodes opcode → control signals |
| `data_memory.v` | Load/store memory with debug access port |
| `gpio.v` | 8-bit GPIO with memory-mapped interface |
| `immediate_generator.v` | Extracts I/S/B/U/J type immediates |
| `instruction_memory.v` | Single-cycle instruction fetch |
| `pc_register.v` | PC with stall support |
| `register_file.v` | 32×32-bit regs with dual-ported read, debug port |
| `timer.v` | Memory-mapped timer with interrupt |
| `riscv_core.v` | Top-level datapath integration |
| `riscv_core_jtag.v` | Core + JTAG TAP wrapper |
| `jtag_tap.v` | IEEE 1149.1 TAP controller |

### Testbench Structure
- `tb/unit/` — Verilog-only unit tests for individual modules
- `tb/integration/` — `tb_rv32i_inline.v` single-file integration test
- `tb/isa/` — `tb_isa_test.v` loads `.mem` files for riscv-tests compliance
- `tb/cocotb/` — Python cocotb tests for JTAG protocol verification
- `tb/uvm/` — pyUVM testbench for JTAG with cocotb entry point

### Cocotb Test Modules (in `tb/cocotb/`)
- `test_tap_fsm` — TAP state machine transitions
- `test_idcode` — IDCODE instruction
- `test_bypass` — BYPASS register
- `test_debug_regs` — GPR read/write via JTAG
- `test_debug_mem` — Memory access via JTAG
- `test_integration` — Full debug session

### ISA Test Memory Files
Generated by `make gen-mem` from riscv-tests, located in `tests/isa/mem/rv32ui-p-*.mem`.
