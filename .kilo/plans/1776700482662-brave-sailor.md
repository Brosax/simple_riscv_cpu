# Simple RISC-V CPU - 项目实现总结

## 1. 核心硬件模块 (RTL)
本项目实现了一个符合 RISC-V RV32I 基础整数指令集规范的单周期、哈佛架构处理器。核心代码位于 `rtl/` 目录，且具备良好的可综合性。主要模块包括：
*   **顶层与控制流**：`riscv_core.v` (CPU顶层), `control_unit.v` (主控制单元), `pc_register.v` (程序计数器)。
*   **运算与寄存器**：`alu.v` & `alu_control_unit.v` (算术逻辑单元及控制), `register_file.v` (32x32bit 寄存器堆), `immediate_generator.v` (立即数生成), `csr_file.v` (控制状态寄存器)。
*   **存储系统**：`instruction_memory.v` (指令存储器), `data_memory.v` (数据存储器，采用 4-bank 结构以正确推断 BRAM)。
*   **内存映射 I/O**：`timer.v` (mtime/mtimecmp定时器及中断), `gpio.v` (通用输入输出控制器)。

## 2. 调试与高级验证基础设施
随着项目的发展，引入了专业的调试接口和现代化的验证环境：
*   **JTAG 调试接口**：新增 `jtag_tap.v` 和 `riscv_core_jtag.v`，支持基于 JTAG 的片上调试。
*   **pyUVM & Cocotb 验证**：在 `tb/uvm/` 和 `tb/cocotb/` 中建立了基于 Python 的 UVM 验证环境，替代了传统的 SystemVerilog UVM，用于验证 JTAG 事务、IDCODE 和状态机。
*   **容器化与 CI/CD**：提供了 Docker 仿真环境（`Dockerfile`, `docker-compose.yml`），并在 GitHub Actions 中配置了自动化流水线，实现代码提交后的自动化构建、仿真并输出波形文件。

## 3. FPGA 部署 (Basys3)
针对 Digilent Basys3 (Xilinx Artix-7) 开发板进行了硬件级集成设计：
*   **硬件接口封装**：新增 `basys3_top.v` 作为 FPGA 顶层模块，包装 RISC-V 核心。
*   **UART 输出支持**：设计了 `uart_tx.v` (115200 bps, 8N1)，将 CPU 写入 `tohost` 地址 (`0x80002000`) 的数据通过串行端口发送至 PC 终端。
*   **引脚约束与外设**：编写了 `constraints/basys3.xdc`，配置了时钟 (12MHz)、UART TX 引脚以及 16 个 LED 状态指示（用于显示正常运行或错误码）。
*   **测试软件**：提供了 C 语言测试程序 (`sw/test_hello.c` -> `test_hello.mem`)，用于在 FPGA 上循环打印 "Hello FPGA!" 以验证全系统工作。

## 4. 测试与验证报告 (Test Report)
项目拥有完善的三级测试体系，确保处理器的可靠性：
*   **第一层：单元测试 (100% 通过)**
    包含 10 个独立测试台 (`tb_alu.v`, `tb_control_unit.v`, `tb_data_memory.v` 等)，覆盖 53 个测试点。
*   **第二层：集成测试 (100% 通过)**
    使用 `tb_rv32i_inline.v`，通过内置的一段包含 ALU、访存、分支和跳转指令的小程序，验证数据通路的协同工作。
*   **第三层：ISA 回归测试 (38/39 通过，通过率 97.4%)**
    运行 RISC-V 官方的 `rv32ui` 一致性测试套件。其中唯一的失败项为 `FENCE.I` 指令，这是由于单周期设计无指令缓存，无需也未实现该可选扩展（属预期内的设计取舍）。

---
**总体结论**：该项目从最初的基础单周期 RV32I 处理器，逐步演进为具有 JTAG 调试能力、UVM 高级验证体系，并能成功在 FPGA (Basys3) 上综合运行、进行 UART 交互的完整系统。
